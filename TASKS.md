## TASK-012：分配策略页面布局优化（Step 1 + 2 合并）

**概述**：将当前三步流程的 Step 1（总览排序）和 Step 2（节奏约束）合并为一页，重新排布区域顺序，并优化排序卡片的展示格式。

**涉及文件**：
- 前端：`PlanAllocateView.vue`（只改动前端）

---

### 改动点

#### 1. 合并 Step 1 + Step 2

当前流程：Step 1（完成日期 + 排序）→ Step 2（折线图 + 节奏 + 上限）→ Step 3（逐项调整）

**改为**：Step 1（折线图 + 完成日期/上限同行 + 节奏 + 排序）→ Step 2（逐项调整）

Step 1 从上到下顺序：
```
① 折线图（WorkloadChart）
② 预计完成日期 + 每日作业上限（同一行/栏）
③ 节奏偏好滑块
④ 预设方案按钮
⑤ 预览统计（可用天数、警告）
⑥ 拖拽排序列表（作业优先级）
   └── 底部按钮：「下一步：逐项调整 →」
```

#### 2. 预计完成日期 + 每日上限放同一栏

当前是分开的两个卡片。改为**同一行两个字段**：

```
┌─────────────────────────────────────┐
│  📅 预计完成   [━━━━●━━━━━━━━━]     │
│  ⏱ 每日上限   [  300  ] 分钟       │
└─────────────────────────────────────┘
```

- 预计完成日期用滑块，显示 `预计 8/25 完成`
- 每日上限用数字输入框 + "分钟" 后缀

#### 3. 排序卡片格式优化

**每张卡片的紧凑格式**：

```
┌──────────────────────────────────────┐
│  ☰ 📐 数学 · 模拟卷  15套 · 22.5h   │
│      📅 7/1 - 7/25 · 隔1天一次       │
└──────────────────────────────────────┘
```

具体改动：
- **总耗时单位改为小时**：`totalAmount * timePerUnit / 60`，保留 1 位小数（如 `22.5h`、`5h`），整数时不显示小数（如 `5h`）
- **日期去年份**：`2026-07-14 ~ 2026-07-25` → 显示为 `7/14 - 7/25`
- **间隔信息**：如果有间隔（`autoInterval > 0`），显示「隔 N+1 天一次」或「每 N+1 天」
- 卡片 padding 减少（`10px 12px` 代替 `12px 14px`）
- 字体适当缩小

#### 4. DateSlice 格式工具函数

新增一个通用格式化函数（可在组件内定义）：

```ts
// "2026-07-14 ~ 2026-07-25" → "7/14 - 7/25"
function dateRangeShort(range: string): string {
  if (!range) return ''
  const parts = range.split(' ~ ')
  return parts.map(d => {
    const m = d.split('-')
    return `${parseInt(m[1])}/${parseInt(m[2])}`
  }).join(' - ')
}

// 分钟 → 小时
function fmtHours(minutes: number): string {
  const h = minutes / 60
  return Number.isInteger(h) ? `${h}h` : `${h.toFixed(1)}h`
}
```
