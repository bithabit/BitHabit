## 一、项目概述

**BitHabit** 是一个帮助学生管理学习计划的 PWA 应用。

- **产品形态**：PWA（Progressive Web App）
- **核心价值**：一键链接分享 → 浏览器打开 → 添加到主屏幕 → 即获得 App 体验，无需安装
- **离线能力**：Service Worker + IndexedDB，断网可打卡，联网自动同步
- **目标用户**：中学生，可分享链接给同班同学共同使用

---

## 二、核心概念：计划（Plan）

### 2.1 定义

> **计划 = 一个独立的行动周期，包含：作业池 + 起止日期 + 日程约束 + 分配偏好**

一个计划就是一次「学习战役」。不同场景创建不同计划，彼此独立。

### 2.2 典型场景

| 场景 | 计划名 | 作业池 | 起止日期 |
|------|--------|--------|----------|
| 暑假 | 暑假作业计划 | 全科暑假作业 | 7/1 → 8/31 |
| 补习班 | 数学补习班作业 | 补习班布置的题 | 7/5 → 7/18 |
| 周末 | 第3周周末计划 | 周末作业 | 9/20 → 9/22 |
| 考试周 | 期末复习计划 | 各科复习卷 | 1/8 → 1/15 |

### 2.3 计划状态

| 状态 | 含义 | 今日页 |
|------|------|--------|
| 活跃 | 正在进行中 | ✅ 显示 |
| 已完成 | 到期且全部完成 | ❌ 归档 |
| 已过期 | 到期但未全部完成 | ❌ 归档（可查看） |

## 三、数据模型 v2

### 3.1 核心关系

```
用户
 └── 计划（Plan）  ← 一个用户有多个计划
      ├── 作业（Homework） ← 作业属于某个计划（不跨计划共享）
      │    └── plan_tasks ← 作业的每日分配实例
      └── 日程约束（Schedule） ← 目前是全局的，后续可考虑归属到计划
```

**关键变更**：作业 `homework` 新增 `plan_id` 外键，作业不再游离在外被计划引用，而是隶属于某个计划。

### 3.2 旧模型 vs 新模型

| | 旧（v1） | 新（v2） |
|---|---------|---------|
| 作业归属 | 游离，被计划引用（多对多） | 属于某个计划（一对多） |
| 创建流程 | 先录作业 → 再生成计划 | 先创建计划 → 在计划内录入作业 |
| 作业列表 | 全局作业池 | 每个计划独立作业列表 |
| 进度 | 跨计划累计 | 本计划内累计 |
| 重新计划 | 全量重生成 | 不需要此概念；新建就是新建 |

### 3.3 数据库需要做的变更

```sql
-- homework 表新增字段
ALTER TABLE homework ADD COLUMN plan_id INT NOT NULL AFTER user_id;
ALTER TABLE homework ADD INDEX idx_plan (plan_id);

-- plan_tasks 表已有 homework_id，保持不变

-- 旧数据迁移：给现有作业归属到某个计划
-- （由 Coder 处理）
```

## 四、核心工作流 v2

### 4.1 创建计划的完整流程

```
① 创建计划
   └── 输入：计划名称、起止日期、每日可用时段、分配策略
   
② 在计划内录入作业（计划创建完成后直接进入）
   └── 手动逐条录入 / AI 自然语言录入
   └── 作业自动归属到当前计划
   
③ 配置日程约束（如需）
   └── 每周固定不可用时段 + 特殊日期
   └── 可复用已有的日程配置，也可微调
   
④ 生成每日任务
   └── 系统按分配策略自动编排
   └── 生成后进入日历视图供查看
   
⑤ 每日微调（高频操作）
   └── 拖拽重排、移动到其他日期、跳过、补做
   └── 打卡确认完成
```

### 4.2 作业录入

作业始终在计划内录入，字段不变：

| 字段 | 说明 | 必填 |
|------|------|------|
| 科目 | 下拉预设 + 可自定义 | ✅ |
| 任务类型 | 练习册/模拟卷/单词/阅读理解/自定义 | ✅ |
| 总量 | 数字 + 单位（页/套/篇/个） | ✅ |
| 单位耗时 | 每单位预估分钟数 | 选填（有默认值） |
| 备注 | 自由文本 | 选填 |

支持：
- 手动逐条录入（快速连录）
- AI 自然语言录入 → DeepSeek API 解析 → 确认保存

### 4.3 日程配置

保持独立页面，全局可复用。当前版本的日程应用于所有计划（后续可考虑归属到计划）。

| 配置项 | 说明 |
|--------|------|
| 每周固定 | 某天某时段不可用（如周一 8:00-14:00 补习班） |
| 特殊日期 | 日期范围 + 时段 + 标签（如 7/15-7/18 全天 旅行） |

### 4.4 分配策略

#### 4.4.1 分配流程（两步）

**Step 1 — 总体策略页**

页面布局（从上到下）：
1. **折线图**（顶部，实时预览）
2. 节奏偏好滑块
3. 预设方案按钮
4. 每日上限设置
5. [下一步：逐项调整] 按钮

**Step 2 — 逐项调整页**

- 作业列表 + 时间窗口约束 + 锁定开关
- 调整后实时刷新上方折线图
- [确认生成] 按钮

#### 4.4.2 节奏偏好

用一个**多档滑块**控制分配节奏：

| 档位 | 标签 | 行为 |
|------|------|------|
| 极左 | 前紧后松 | 早期集中完成，后期轻松 |
| 左 | 偏紧 | 略偏向早期 |
| 中 | 均匀 | 每天分配量接近 |
| 右 | 偏松 | 略偏向后期 |
| 极右 | 前松后紧 | 后期集中完成 |

滑块实时影响分配权重曲线，折线图同步刷新。

预设方案按钮直接跳转到对应档位：
- [均匀] [前紧后松] [前松后紧]

#### 4.4.3 每日工作量折线图

- X 轴：日期（计划起止范围内）
- Y 轴：预估分钟数
- 虚线：每日上限线
- 超出上限的柱子变红（预警）
- 点击某天可查看该天作业明细
- 折线图放在页面上方，策略调整实时刷新

#### 4.4.3a 折线图动画

| 动画 | 触发时机 | 效果 | 时长 | 缓动 |
|------|---------|------|------|------|
| 入场 Stagger | 页面首次加载 | 柱子从左到右逐个弹入（delay = index × 40ms）| 500ms/柱 | spring |
| 值过渡 | 滑块/上限变化 | 柱子平滑迁移到新 Y 值和高度 | 400ms | spring |
| 超限脉冲 | 某天超过上限 | 柱子变红 + 轻微脉冲缩放（1→1.05→1）| 300ms | smooth |

实现方式：SVG `<rect>` 增加 CSS `transition`（y/height/fill），复用项目 `--ease-spring` / `--ease-smooth` Token。入场 stagger 通过首次渲染 bars 高度为 0，下一帧赋值真实高度触发 transition。

柱状图的柱子在 `<template>` 中加 `class="bar-rect"`，配合 CSS：
```css
rect.bar-rect {
  transition: y 0.4s var(--ease-spring),
              height 0.4s var(--ease-spring),
              fill 0.3s var(--ease-smooth);
}
```

#### 4.4.4 每日上限

- 默认值：**300 分钟**（5 小时）
- 用户可调整，输入整数分钟数
- 上限不是硬拒绝，而是预警 + 自动调整：
  - 超限的作业优先挪到相邻有空的天
  - 所有天都满了则提示「建议延长计划结束日期或减少作业量」

#### 4.4.5 逐项调整（时间窗口约束）

在逐项调整页，每项作业显示：

| 字段 | 说明 |
|------|------|
| 作业名 | 科目·类型 · 总量 |
| 起始日 | 从哪天开始分配（默认计划起始日） |
| 截止日 | 在哪天前完成（默认计划结束日） |
| 锁定 | 开关，锁定后不参与重新分配 |

约束逻辑：
- 起始日 ≥ 计划起始日，截止日 ≤ 计划结束日
- 起始日 ≤ 截止日
- 调整后系统重新编排，但遵守时间窗口

**不是逐日编辑**——系统仍然负责每天具体分配，用户只表达偏好。

#### 4.4.6 分配算法（后端）

1. 计算可用天数 = 起止日期内 - 日程约束 - 休息日
2. 按节奏偏好生成权重曲线（每天权重 w₁, w₂, ..., wₙ）
3. 对每项作业：
   - 有时间窗口约束的，只在窗口内分配
   - 按权重比例分配单位，整数取整，余数按权重排序分配
4. 每天总工时 ≤ 每日上限（超限自动向相邻日期溢出）
5. 锁定的作业跳过重新分配

### 4.5 今日任务页

合并显示所有**活跃计划**的今日任务：

```
今日任务
├── 📘 暑假作业计划
│   ├── 数学·模拟卷  1 套 · 90 分钟
│   └── 英语·阅读理解 2 篇 · 30 分钟
├── 📙 数学补习班
│   └── 习题集  3 页 · 30 分钟
└── 总计：3 项 · 约 150 分钟
```

按计划分组，用颜色或标签区分来源。

### 4.6 日历视图

- 月视图 + 日弹窗（已有）
- 日弹窗内按计划分组显示任务
- 拖拽重排、移动日期（TASK-005 规格）
- 不同计划的任务可在同一天并存

## 六、作业进度条与快捷操作

进度条和快捷操作改到**计划详情页的作业列表**中展示（而非全局作业页，因为全局作业页已经不存在了）。

每张作业卡片：

**已生成每日任务时：**
- 进度条：已完成单位 / 总单位，绿色填充
- ⋯ 菜单：[编辑] [删除]

**尚未生成每日任务时：**
- 提示「尚未生成计划，请先生成」
- ⋯ 菜单：[编辑] [删除]

不再需要「添加到计划」和「重新计划」按钮——作业天然属于计划。
