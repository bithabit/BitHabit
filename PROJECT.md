## 一、项目概述

**BitHabit** 是一个帮助学生管理学习计划的 PWA 应用。

- **产品形态**：PWA（Progressive Web App）
- **核心价值**：一键链接分享 → 浏览器打开 → 添加到主屏幕 → 即获得 App 体验，无需安装
- **离线能力**：Service Worker + IndexedDB，断网可打卡，联网自动同步
- **目标用户**：中学生，可分享链接给同班同学共同使用

## 二、核心概念：计划（Plan）

### 2.1 定义

> **计划 = 一个独立的行动周期，包含：作业池 + 起止日期 + 日程约束 + 分配偏好**

一个计划就是一次「学习战役」。不同场景创建不同计划，彼此独立。

### 2.2 典型场景

| 场景 | 计划名 | 作业池 | 起止日期 |
|------|--------|--------|----------|
| 暑假 | 暑假作业计划 | 全科暑假作业 | 7/1 → 8/31 |
| 补习班 | 数学补习班作业 | 补习班布置的题 | 7/5 → 7/18 |
| 周末 | 第3周周末计划 | 周末作业 | 9/20 → 9/22 |
| 考试周 | 期末复习计划 | 各科复习卷 | 1/8 → 1/15 |

### 2.3 计划状态

| 状态 | 含义 | 今日页 |
|------|------|--------|
| 活跃 | 正在进行中 | ✅ 显示 |
| 已完成 | 到期且全部完成 | ❌ 归档 |
| 已过期 | 到期但未全部完成 | ❌ 归档（可查看） |

## 三、数据模型 v2

### 3.1 核心关系

```
用户
 └── 计划（Plan）  ← 一个用户有多个计划
      ├── 作业（Homework） ← 作业属于某个计划（不跨计划共享）
      │    └── plan_tasks ← 作业的每日分配实例
      └── 日程约束（Schedule） ← 目前是全局的，后续可考虑归属到计划
```

**关键变更**：作业 `homework` 新增 `plan_id` 外键，作业不再游离在外被计划引用，而是隶属于某个计划。

### 3.2 旧模型 vs 新模型

| | 旧（v1） | 新（v2） |
|---|---------|---------|
| 作业归属 | 游离，被计划引用（多对多） | 属于某个计划（一对多） |
| 创建流程 | 先录作业 → 再生成计划 | 先创建计划 → 在计划内录入作业 |
| 作业列表 | 全局作业池 | 每个计划独立作业列表 |
| 进度 | 跨计划累计 | 本计划内累计 |
| 重新计划 | 全量重生成 | 不需要此概念；新建就是新建 |

### 3.3 数据库需要做的变更

```sql
-- homework 表新增字段
ALTER TABLE homework ADD COLUMN plan_id INT NOT NULL AFTER user_id;
ALTER TABLE homework ADD INDEX idx_plan (plan_id);
ALTER TABLE homework ADD COLUMN window_start DATE DEFAULT NULL AFTER time_per_unit;
ALTER TABLE homework ADD COLUMN window_end DATE DEFAULT NULL AFTER window_start;
ALTER TABLE homework ADD COLUMN locked TINYINT(1) DEFAULT 0 AFTER window_end;
ALTER TABLE homework ADD COLUMN priority INT DEFAULT 0 AFTER locked;           -- v3: 优先级排序
ALTER TABLE homework ADD COLUMN interval_days INT DEFAULT 0 AFTER priority;   -- v3: 间隔天数（0=每天，1=隔天，2=隔两天…）
ALTER TABLE homework ADD COLUMN allocated_range VARCHAR(50) DEFAULT NULL AFTER interval_days;  -- v3: 预估分配时间范围（只读快照）
```

- `plan_id`：作业归属到计划（v2 核心变更）
- `window_start`：该项作业最早开始日期，NULL=计划起始日
- `window_end`：该项作业最晚完成日期，NULL=计划结束日
- `locked`：1=锁定不参与重新分配
- `priority`：用户拖拽排序的优先级值（越小越优先分配，0 最高）
- `interval_days`：间隔天数，0=每天分配，1=隔 1 天（即每 2 天一次），2=隔 2 天（每 3 天一次）……
- `allocated_range`：上次生成时写入的预估时间范围（如 "7/10-7/20"），只读展示

## 四、核心工作流 v2

### 4.1 创建计划的完整流程

```
① 创建计划
   └── 输入：计划名称、起止日期、每日可用时段、分配策略
   
② 在计划内录入作业（计划创建完成后直接进入）
   └── 手动逐条录入 / AI 自然语言录入
   └── 作业自动归属到当前计划
   
③ 配置日程约束（如需）
   └── 每周固定不可用时段 + 特殊日期
   └── 可复用已有的日程配置，也可微调
   
④ 生成每日任务
   └── 系统按分配策略自动编排
   └── 生成后进入日历视图供查看
   
⑤ 每日微调（高频操作）
   └── 拖拽重排、移动到其他日期、跳过、补做
   └── 打卡确认完成
```

### 4.2 作业录入

作业始终在计划内录入，字段不变：

| 字段 | 说明 | 必填 |
|------|------|------|
| 科目 | 下拉预设 + 可自定义 | ✅ |
| 任务类型 | 练习册/模拟卷/单词/阅读理解/自定义 | ✅ |
| 总量 | 数字 + 单位（页/套/篇/个） | ✅ |
| 单位耗时 | 每单位预估分钟数 | 选填（有默认值） |
| 备注 | 自由文本 | 选填 |

支持：
- 手动逐条录入（快速连录）
- AI 自然语言录入 → DeepSeek API 解析 → 确认保存

### 4.3 日程配置

保持独立页面，全局可复用。当前版本的日程应用于所有计划（后续可考虑归属到计划）。

| 配置项 | 说明 |
|--------|------|
| 每周固定 | 某天某时段不可用（如周一 8:00-14:00 补习班） |
| 特殊日期 | 日期范围 + 时段 + 标签（如 7/15-7/18 全天 旅行） |

### 4.4 分配策略（v3）

#### 4.4.1 分配流程（三步）

**Step 1 — 总览 & 优先级排序（新增）**

页面布局（从上到下）：
1. 计划基本信息摘要（起止日期、可用天数）
2. **预计完成日期行**：显示「📅 预计 8/25 完成全部作业」，旁边有个小滑块可微调（往前=更紧凑，往后=更宽松）← **新增**
3. **作业优先级拖拽排序列表**（核心新功能）
   - 每行：拖拽手柄 + 科目·类型 + 总量·总耗时
   - 可上下拖拽重新排序（物理排第一 → 物理的作业优先被安排）
   - 每行下方显示预估时间范围 + 推荐间隔（只读小字，如 `📅 预计 7/14 - 7/28 · 隔10天一篇`）
   - 拖拽后 1 秒防抖自动触发预览，刷新预估
4. [下一步：节奏与约束] 按钮

> **预计完成日期逻辑**：
> - 由算法算出最后一天有作业的日期
> - 用户可滑动滑块调早（提前完成）或调晚（推迟完成）
> - 调早 → 系统压缩节奏（增加每日分配量）
> - 调晚 → 系统拉长节奏（减少每日分配量）
> - 折线图标注此线（竖虚线），清晰展示「最后一天有作业」

**Step 2 — 节奏 & 约束（原 Step 1 扩展）**

页面布局（从上到下）：
1. **折线图**（顶部，实时预览）← **新增「预计完成线」竖虚线**
2. 节奏偏好滑块（5 档 + 3 预设按钮）
3. 每日上限设置
4. [下一步：逐项调整] 按钮

**Step 3 — 逐项调整（原 Step 2 扩展）**

- 作业列表，每项显示：
  - 科目·类型 · 总量（header）
  - 起始日滑块 + 手动输入框（双向绑定）← **新增滑块**
  - 截止日滑块 + 手动输入框（双向绑定）← **新增滑块**
  - **间隔天数** 滑块（自动推荐值 + 可手动调整）← **新增字段，自动推荐**
  - 锁定开关
- 间隔天数自动推荐逻辑：`Math.round(窗口天数 / 总单位数)`，即为保证均匀覆盖自动算出的间隔
- 调整后实时刷新上方折线图
- [确认生成] 按钮

#### 4.4.2 分配算法 v3

进入 Step 2 后的分配算法是一个**多阶段处理流程**：

```
阶段 0：预处理
  - 收集所有作业（含优先级 priority、时间窗口、间隔 interval_days）
  - 按 priority 升序排列（priority 越小先分配）
  - 计算可用天数列表（排除了日程约束后）

阶段 1：间隔过滤
  - 对每项作业：
    - interval_days = 0 → 窗口内所有天都可用
    - interval_days = k → 窗口内每隔 k 天取一天（从窗口起始日算起）
    - 如果有时间窗口约束，窗口起始日优先作为间隔起点

阶段 2：自动间隔推荐
  - 对每项作业：
    - 窗口内的有效天数 = M
    - 如果 interval_days 未手动设置（= 初始值 0）：
      - 自动推荐间隔 = max(0, floor(M / totalAmount) - 1)
        → 即均匀覆盖窗口，保证 totalAmount 单位分布均匀
        例：5 篇作文 / 60 天 → floor(60/5) - 1 = 11 → 每 12 天一篇
        例：60 页练习册 / 60 天 → floor(60/60) - 1 = 0 → 每天 1 页
    - 如果 interval_days 已手动设置（≠ 0），直接使用用户设定的值
    - 按 interval_days 过滤可用天集合
  - 用户可在 Step 3 中手动调整间隔，覆盖自动推荐

阶段 3：节奏权重
  - 与 v2 相同：按 rhythm(-2..2) 生成每日权重曲线 w(t)
  - 权重应用于阶段 2 确定的有效天数集合

阶段 4：按优先级顺序分配
  - 按 priority 从高到低（数字从小到大）依次分配每项作业
  - 高优先级的作业先占满每天的容量
  - 后分配的作业在剩余容量中寻找合适的天
  - 这自然实现了「先写物理，再写化学」

阶段 5：整数取整
  - 与 v2 相同：按权重比例取整，余数按权重排序分配

阶段 6：每日上限溢出处理
  - 优先级低的任务先被挤到相邻天
  - 总量小的任务（如作文）倾向于被挤到后期天（因为它在优先级排序中靠后，早期天已被高优任务占满）

阶段 7：科目轮转（可选）
  - 每天的任务按科目排序展示
  - 不强制每天一定有每个科目（能力有限，由数量决定）
  - 间隔分配 + 科目多样性天然形成交错效果
```

#### 4.4.3 自动间隔推荐

对于总量远小于窗口天数的任务（如「作文 5 篇」横跨 60 天），v2 算法会把 5 篇均匀散到 60 天里——看起来像「每天分配 0.08 篇」不合理。

v3 的改进：
- 系统自动计算推荐间隔：`floor(窗口天数 / 总量) - 1`
- 作文 5 篇 / 60 天 → floor(60/5)-1 = 10 → 每 11 天写一篇
- 数学 60 页 / 60 天 → floor(60/60)-1 = 0 → 每天 1 页
- 用户可在 Step 3 手动调整间隔覆盖自动推荐
- Step 1 每行下方展示推荐间隔（如「隔 10 天一篇」）

#### 4.4.4 节奏偏好（不变）

用一个**多档滑块**控制分配节奏：

| 档位 | 标签 | 行为 |
|------|------|------|
| 极左 | 前紧后松 | 早期集中完成，后期轻松 |
| 左 | 偏紧 | 略偏向早期 |
| 中 | 均匀 | 每天分配量接近 |
| 右 | 偏松 | 略偏向后期 |
| 极右 | 前松后紧 | 后期集中完成 |

#### 4.4.5 每日工作量折线图（不变）

- X 轴：日期（计划起止范围内）
- Y 轴：预估分钟数
- 虚线：每日上限线
- 超出上限的柱子变红（预警）
- 点击某天可查看该天作业明细

#### 4.4.5a 折线图动画（不变）

| 动画 | 触发时机 | 效果 | 时长 | 缓动 |
|------|---------|------|------|------|
| 入场 Stagger | 页面首次加载 | 柱子从左到右逐个弹入 | 500ms/柱 | spring |

#### 4.4.6 逐项调整控件升级

| 字段 | v2 控件 | v3 控件 |
|------|---------|---------|
| 起始日 | `<input type="date">` | **滑块** (`<input type="range">`) **+ 手动输入框**，双向绑定 |
| 截止日 | `<input type="date">` | **滑块** (`<input type="range">`) **+ 手动输入框**，双向绑定 |
| 锁定 | toggle 开关 | toggle 开关（不变） |
| 间隔 | — | **新增滑块** (`<input type="range">` min=0 max=间隔上限)，标签「每 N+1 天一次」 |

滑块设计：
- 起始日滑块：范围 = [计划起始日, 截止日-1]，Step=1天
- 截止日滑块：范围 = [起始日+1, 计划结束日]，Step=1天
- 间隔滑块：范围 = [0, 窗口天数/2]（最大不超过窗口的一半）
- 每个滑块旁边一个小的数字输入框，可手动输入精确日期/间隔值
- 滑块和输入框双向绑定：改任一，另一同步
