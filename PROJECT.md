## 一、项目概述

**BitHabit** 是一个帮助学生管理学习计划的 PWA 应用。

- **产品形态**：PWA（Progressive Web App）
- **核心价值**：一键链接分享 → 浏览器打开 → 添加到主屏幕 → 即获得 App 体验，无需安装
- **离线能力**：Service Worker + IndexedDB，断网可打卡，联网自动同步
- **目标用户**：中学生，可分享链接给同班同学共同使用

## 二、核心概念：计划（Plan）

### 2.1 定义

> **计划 = 一个独立的行动周期，包含：作业池 + 起止日期 + 日程约束 + 分配偏好**

一个计划就是一次「学习战役」。不同场景创建不同计划，彼此独立。

### 2.2 典型场景

| 场景 | 计划名 | 作业池 | 起止日期 |
|------|--------|--------|----------|
| 暑假 | 暑假作业计划 | 全科暑假作业 | 7/1 → 8/31 |
| 补习班 | 数学补习班作业 | 补习班布置的题 | 7/5 → 7/18 |
| 周末 | 第3周周末计划 | 周末作业 | 9/20 → 9/22 |
| 考试周 | 期末复习计划 | 各科复习卷 | 1/8 → 1/15 |

### 2.3 计划状态

| 状态 | 含义 | 今日页 |
|------|------|--------|
| 活跃 | 正在进行中 | ✅ 显示 |
| 已完成 | 到期且全部完成 | ❌ 归档 |
| 已过期 | 到期但未全部完成 | ❌ 归档（可查看） |

## 三、数据模型 v2

### 3.1 核心关系

```
用户
 └── 计划（Plan）  ← 一个用户有多个计划
      ├── 作业（Homework） ← 作业属于某个计划（不跨计划共享）
      │    └── plan_tasks ← 作业的每日分配实例
      └── 日程约束（Schedule） ← 目前是全局的，后续可考虑归属到计划
```

**关键变更**：作业 `homework` 新增 `plan_id` 外键，作业不再游离在外被计划引用，而是隶属于某个计划。

### 3.2 旧模型 vs 新模型

| | 旧（v1） | 新（v2） |
|---|---------|---------|
| 作业归属 | 游离，被计划引用（多对多） | 属于某个计划（一对多） |
| 创建流程 | 先录作业 → 再生成计划 | 先创建计划 → 在计划内录入作业 |
| 作业列表 | 全局作业池 | 每个计划独立作业列表 |
| 进度 | 跨计划累计 | 本计划内累计 |
| 重新计划 | 全量重生成 | 不需要此概念；新建就是新建 |

### 3.3 数据库需要做的变更

```sql
-- homework 表新增字段
ALTER TABLE homework ADD COLUMN plan_id INT NOT NULL AFTER user_id;
ALTER TABLE homework ADD INDEX idx_plan (plan_id);

-- plan_tasks 表已有 homework_id，保持不变

-- 旧数据迁移：给现有作业归属到某个计划
-- （由 Coder 处理）
```

## 四、核心工作流 v2

### 4.1 创建计划的完整流程

```
① 创建计划
   └── 输入：计划名称、起止日期、每日可用时段、分配策略
   
② 在计划内录入作业（计划创建完成后直接进入）
   └── 手动逐条录入 / AI 自然语言录入
   └── 作业自动归属到当前计划
   
③ 配置日程约束（如需）
   └── 每周固定不可用时段 + 特殊日期
   └── 可复用已有的日程配置，也可微调
   
④ 生成每日任务
   └── 系统按分配策略自动编排
   └── 生成后进入日历视图供查看
   
⑤ 每日微调（高频操作）
   └── 拖拽重排、移动到其他日期、跳过、补做
   └── 打卡确认完成
```

### 4.2 作业录入

作业始终在计划内录入，字段不变：

| 字段 | 说明 | 必填 |
|------|------|------|
| 科目 | 下拉预设 + 可自定义 | ✅ |
| 任务类型 | 练习册/模拟卷/单词/阅读理解/自定义 | ✅ |
| 总量 | 数字 + 单位（页/套/篇/个） | ✅ |
| 单位耗时 | 每单位预估分钟数 | 选填（有默认值） |
| 备注 | 自由文本 | 选填 |

支持：
- 手动逐条录入（快速连录）
- AI 自然语言录入 → DeepSeek API 解析 → 确认保存

### 4.3 日程配置

保持独立页面，全局可复用。当前版本的日程应用于所有计划（后续可考虑归属到计划）。

| 配置项 | 说明 |
|--------|------|
| 每周固定 | 某天某时段不可用（如周一 8:00-14:00 补习班） |
| 特殊日期 | 日期范围 + 时段 + 标签（如 7/15-7/18 全天 旅行） |

### 4.4 分配策略

v1：平均分配（整数单位）

1. 计算可用天数 = 起止日期内 - 日程约束占用的天数 - 休息日
2. 每科作业：dailyUnits = floor(totalAmount / availableDays)，余数前 N 天多 1 单位
3. 每天总工时 ≤ 每日可用时段上限
4. 超限自动延后到有容量的日期

### 4.5 今日任务页

合并显示所有**活跃计划**的今日任务：

```
今日任务
├── 📘 暑假作业计划
│   ├── 数学·模拟卷  1 套 · 90 分钟
│   └── 英语·阅读理解 2 篇 · 30 分钟
├── 📙 数学补习班
│   └── 习题集  3 页 · 30 分钟
└── 总计：3 项 · 约 150 分钟
```

按计划分组，用颜色或标签区分来源。

### 4.6 日历视图

- 月视图 + 日弹窗（已有）
- 日弹窗内按计划分组显示任务
- 拖拽重排、移动日期（TASK-005 规格）
- 不同计划的任务可在同一天并存

## 六、作业进度条与快捷操作

进度条和快捷操作改到**计划详情页的作业列表**中展示（而非全局作业页，因为全局作业页已经不存在了）。

每张作业卡片：

**已生成每日任务时：**
- 进度条：已完成单位 / 总单位，绿色填充
- ⋯ 菜单：[编辑] [删除]

**尚未生成每日任务时：**
- 提示「尚未生成计划，请先生成」
- ⋯ 菜单：[编辑] [删除]

不再需要「添加到计划」和「重新计划」按钮——作业天然属于计划。
